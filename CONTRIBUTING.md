---

## PR Guidelines

If submitting a PR:

1. **Open an issue first** — discuss before coding
2. **Keep changes focused** — one feature per PR
3. **Write clear descriptions** — what and why
4. **Test locally** — don't break things
5. **Follow existing style** — match the codebase

---

## Issue Guidelines

Good issue = actionable issue.

### Bug Reports
Include:
- What you expected
- What actually happened
- Steps to reproduce
- Screenshots (if applicable)

### Feature Requests
Include:
- Problem you're solving
- Proposed solution
- Alternatives considered
- Phase this fits into

### Questions
Use Discussions (not Issues) for general questions.

---

## Code of Conduct

### Be Respectful
Disagreements are fine. Personal attacks are not.

### Be Constructive
Bring solutions, not just problems.

### Be Inclusive
Satan6x6 is for everyone. Discrimination of any kind is not tolerated.

### No Spam
- No shilling unrelated projects
- No scam links
- No airdrop farming behavior
